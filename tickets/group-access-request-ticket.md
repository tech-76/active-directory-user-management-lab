# Ticket: Security Group Access Request

## Ticket Summary

User requested access to a shared department folder.

## User Impact

User could not access files required for their department work.

## Reported Issue

User received an access denied message when opening a shared network folder.

## Troubleshooting Steps

1. Confirmed the shared folder path.
2. Confirmed user account.
3. Reviewed access request.
4. Confirmed manager approval.
5. Identified the correct security group for the folder.
6. Added user to the approved security group.
7. Asked user to sign out and sign back in.
8. Confirmed user could access the shared folder.

## Findings

User was not a member of the required security group.

## Resolution

Added user to the approved security group after manager approval. User confirmed access was working.

## Final Ticket Note

Verified approval, added user to required security group, user signed out and back in, and shared folder access was confirmed.
