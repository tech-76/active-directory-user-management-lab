# Active Directory Overview

## Purpose

Active Directory is commonly used by organizations to manage users, computers, groups, and access to network resources. Help desk and IT support teams often work with Active Directory when supporting password resets, account lockouts, user onboarding, group access, and account changes.

## What Active Directory Manages

Active Directory can manage:

- User accounts
- Computer accounts
- Security groups
- Distribution groups
- Organizational Units
- Group Policy links
- Access permissions
- Authentication and sign-in access

## Common Help Desk Tasks

- Create new user accounts
- Reset user passwords
- Unlock accounts
- Disable accounts
- Update user information
- Add or remove users from groups
- Check account status
- Confirm user location in an OU
- Support onboarding and offboarding

## Important Terms

| Term | Meaning |
|---|---|
| Domain | A managed network environment |
| Domain Controller | Server that handles authentication and directory services |
| User Account | Identity used by a person to sign in |
| Computer Account | Identity used by a domain-joined computer |
| Security Group | Group used to assign access or permissions |
| Organizational Unit | Container used to organize users, computers, and groups |
| Group Policy | Centralized settings applied to users or computers |

## Security Notes

- Do not create or modify accounts without approval.
- Use least privilege when assigning access.
- Document all changes in a ticket.
- Never store passwords in tickets.
- Escalate suspicious account activity.
- Follow company onboarding and offboarding procedures.

## Example Ticket Note

Reviewed Active Directory account status for user. Account was active, placed in the correct OU, and assigned to the approved security groups. No unauthorized changes were made.
