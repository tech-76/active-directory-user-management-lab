# Security Groups Basics

## Purpose

Security groups are used to organize users and assign access to resources. Help desk teams may add or remove users from groups after approval.

## What Security Groups Are Used For

Security groups may control access to:

- Shared folders
- Printers
- Applications
- VPN access
- Department resources
- Distribution or collaboration resources
- Administrative tools

## Common Group Tasks

- Add user to an approved group
- Remove user from a group
- Confirm current group membership
- Check whether access is inherited through a group
- Document group access changes
- Escalate unusual access requests

## Access Request Workflow

1. Review the request.
2. Confirm the resource needed.
3. Confirm the security group name.
4. Confirm manager or owner approval.
5. Add the user to the approved group.
6. Ask user to sign out and back in if needed.
7. Confirm access works.
8. Document the change in the ticket.

## Least Privilege

Users should only receive the access required for their job. Avoid adding users to broad groups unless approved.

## Common Mistakes to Avoid

- Adding users to the wrong group
- Granting access without approval
- Adding access directly instead of using a group
- Leaving old access after job changes
- Not documenting the reason for access

## Example Ticket Note

Verified manager approval for shared folder access. Added user to the approved security group and advised user to sign out and back in. User confirmed access was working.
