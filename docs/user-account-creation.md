# User Account Creation

## Purpose

User account creation is a common onboarding task. A clear process helps ensure the user receives the correct access while keeping the environment secure.

## Required Information

Before creating a user account, confirm:

- Full legal name
- Preferred display name
- Job title
- Department
- Manager
- Start date
- Username format
- Email address format
- Required groups
- Required shared resources
- Required applications
- Approval from manager or authorized requester

## Standard Account Creation Workflow

1. Review the new user request.
2. Confirm approval.
3. Confirm required user details.
4. Create the user account using the correct naming standard.
5. Place the account in the correct Organizational Unit.
6. Set a temporary password according to policy.
7. Require password change at first sign-in.
8. Add the user to approved security groups.
9. Confirm account status.
10. Document the completed setup in the ticket.

## Naming Standards

Organizations may use different naming formats, such as:

- first.last
- firstinitiallastname
- employee ID
- department-based naming

Always follow company policy.

## Common Mistakes to Avoid

- Creating duplicate accounts
- Assigning access without approval
- Placing the user in the wrong OU
- Adding the user to too many groups
- Sending passwords through insecure channels
- Forgetting to require password change at first sign-in
- Leaving the account enabled before the start date if policy does not allow it

## Example Ticket Note

Created new Active Directory user account for approved new hire. Account placed in the correct department OU, temporary password set according to policy, and user added to approved security groups.
