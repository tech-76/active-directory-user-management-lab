# Common Active Directory Tasks

## Purpose

This document summarizes common Active Directory tasks handled by IT Support, Service Desk, Desktop Support, and Junior Systems Administrator teams.

## Task: Create New User

### Steps

1. Confirm request approval.
2. Confirm user details.
3. Create user account.
4. Place user in correct OU.
5. Set temporary password.
6. Require password change at next sign-in.
7. Add approved groups.
8. Document completion.

## Task: Reset Password

### Steps

1. Verify user identity.
2. Confirm account is active.
3. Reset password.
4. Require password change at next sign-in.
5. Confirm successful sign-in.
6. Document ticket.

## Task: Unlock Account

### Steps

1. Verify user identity.
2. Confirm account lockout.
3. Unlock account.
4. Ask about recent password changes.
5. Check for saved old credentials.
6. Confirm user can sign in.
7. Monitor for repeat lockout.

## Task: Add User to Group

### Steps

1. Confirm request approval.
2. Confirm group name.
3. Confirm user account.
4. Add user to group.
5. Ask user to sign out and back in if required.
6. Confirm access.
7. Document change.

## Task: Disable User Account

### Steps

1. Confirm offboarding request approval.
2. Confirm user account.
3. Disable account.
4. Remove or review group memberships if required.
5. Move account to disabled OU if policy requires it.
6. Document completion.

## Task: Update User Details

### Steps

1. Confirm requested change.
2. Verify approval if needed.
3. Update details such as job title, department, or manager.
4. Confirm changes.
5. Document ticket.

## Good Ticket Notes Should Include

- User affected
- Request reason
- Approval source
- Action completed
- Security considerations
- User confirmation
