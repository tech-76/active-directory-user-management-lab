# Password Reset Process

## Purpose

Password resets are one of the most common help desk tasks. The process must restore user access while protecting the account from unauthorized changes.

## Common Reasons for Password Reset

- User forgot password
- Password expired
- User cannot sign in
- Account locked after failed attempts
- New user needs first-time password
- User suspects password compromise

## Password Reset Workflow

1. Verify the user's identity.
2. Review the ticket or request.
3. Confirm the account is active.
4. Check if the account is locked.
5. Reset the password if approved.
6. Require password change at next sign-in.
7. Ask the user to test sign-in.
8. Confirm access is restored.
9. Document the action in the ticket.

## Identity Verification Examples

Depending on company policy, verification may include:

- Employee ID
- Manager confirmation
- Approved ticket request
- Phone call to a known company number
- Security questions
- HR record confirmation

## Security Considerations

- Do not reset passwords without verifying identity.
- Do not send passwords through plain email or chat unless company policy allows a secure method.
- Do not reuse temporary passwords.
- Escalate possible account compromise.
- Record the reset in the ticket.
- Do not include the actual password in ticket notes.

## Example Resolution Note

Verified user identity according to support procedure. Reset Active Directory password, required password change at next sign-in, and confirmed user could log in successfully.
