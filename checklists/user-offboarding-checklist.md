# User Offboarding Checklist

## Purpose

Use this checklist when documenting an approved user offboarding request.

## Request Verification

- [ ] Offboarding request received
- [ ] Approval confirmed
- [ ] User account confirmed
- [ ] Last working date confirmed
- [ ] Manager confirmed
- [ ] Required access removal confirmed

## Account Actions

- [ ] User account disabled
- [ ] Password reset if policy requires it
- [ ] Account moved to disabled OU if required
- [ ] Group memberships reviewed
- [ ] Remote access removed if applicable
- [ ] Shared resource access removed if required

## Additional Access Review

- [ ] VPN access reviewed
- [ ] Email access reviewed
- [ ] Application access reviewed
- [ ] Shared mailbox access reviewed
- [ ] Admin rights reviewed
- [ ] Device return process confirmed if applicable

## Documentation

- [ ] Ticket updated with completed actions
- [ ] No unnecessary personal data included
- [ ] Manager notified
- [ ] Escalation completed if needed

## Security Notes

- Offboarding should be completed on time.
- Do not delete accounts unless policy requires it.
- Preserve records according to company procedure.
- Escalate if privileged access is involved.
